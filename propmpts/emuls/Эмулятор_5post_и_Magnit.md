Для magnit - https://magnit-emul-api.vkus.online

Реализовать эмуляцию только работы с заказами и их статусами: создание заказа в их системе, проверка статуса. Все остальные функции типа синхронизации ПВЗ, авторизации и прочего - не надо. 

Хранить данные можно в тоже Postgres-базе что и основная база, только добавь префикс к таблицам какой-нибудь префикс, помогающий легко отличить таблицы для эмулятора от основных таблиц базы. 

Для управления заказами в эмуляторах напиши небольшую интерактивную CLI-админку для управления - главная задача - двигать статусы заказов по life-cycle соответствующему вендору (для 5post- свой, а для magnit-свой)
