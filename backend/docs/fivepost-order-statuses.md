# 5Post API — Статусы заказов

Источник: API `GET /api/v1/orders/{order_id}/status`, маппинг из `packages/integrations/fivepost/utils.py`

## Все статусы (12)

| Статус | Название (RU) | Что происходит в реальности |
|---|---|---|
| `NEW` | Новый | Заказ только что создан через API. Посылка ещё не передана курьеру 5Post. Отправитель печатает этикетку и готовит посылку. |
| `CREATED` | Создан | Заказ зарегистрирован в системе 5Post и ожидает физической передачи посылки. Данные валидированы. |
| `ACCEPTED` | Принят | Посылка физически принята на складе 5Post (курьер забрал или отправитель привёз на склад). Начинается обработка. |
| `SORTING` | На сортировке | Посылка на сортировочном центре 5Post. Определяется маршрут доставки до конечного ПВЗ/постамата. |
| `IN_TRANSIT` | В пути | Посылка в дороге — едет на транспорте 5Post от сортировочного центра до города/района получателя. |
| `ON_POINT` | Прибыл в ПВЗ | Посылка доставлена в пункт выдачи (постамат или кассу Пятёрочки), но ещё не размещена в ячейке. Идёт приёмка. |
| `READY_FOR_PICKUP` | Готов к выдаче | Посылка размещена в ячейке постамата или на полке ПВЗ. Клиент получил SMS/уведомление с кодом для получения. Срок хранения обычно 5-7 дней. |
| `ISSUED` | Выдан | Клиент забрал посылку. Ввёл код на постамате или получил на кассе Пятёрочки. Доставка завершена успешно. |
| `DELIVERED` | Доставлен | Синоним `ISSUED`. Используется в некоторых версиях API. Посылка выдана клиенту. |
| `RETURNING` | Возврат | Клиент не забрал посылку в срок хранения. Посылка снята с ПВЗ и отправлена обратно на склад отправителя. |
| `RETURNED` | Возвращён | Посылка вернулась на склад отправителя (нас). Нужно решить: повторная отправка или возврат денег клиенту. |
| `CANCELLED` | Отменён | Заказ отменён до отправки (партнёром через API или автоматически системой). Посылка не была передана в доставку. |
| `LOST` | Утерян | Посылка потеряна в процессе доставки. Редкий статус. Требуется обращение в поддержку 5Post и компенсация клиенту. |

## Жизненный цикл

### Нормальная доставка (3-7 дней)
```
NEW → CREATED → ACCEPTED → SORTING → IN_TRANSIT → ON_POINT → READY_FOR_PICKUP → ISSUED
 │       │          │          │          │            │              │               │
 │       │          │          │          │            │              │               └─ Клиент забрал
 │       │          │          │          │            │              └─ Лежит в ячейке, ждёт клиента
 │       │          │          │          │            └─ Привезли в ПВЗ, приёмка
 │       │          │          │          └─ Едет в фуре/машине
 │       │          │          └─ Сортировочный центр
 │       │          └─ Курьер забрал у нас
 │       └─ Система приняла заказ
 └─ Мы создали через API
```

### Возврат (клиент не забрал)
```
READY_FOR_PICKUP → (срок хранения истёк) → RETURNING → RETURNED
```

### Исключения
- `CANCELLED` — мы отменили заказ до передачи курьеру
- `LOST` — посылка потеряна (страховой случай)

## Что видит клиент на трекинг-странице

| 5Post статус | Что показываем клиенту |
|---|---|
| `NEW`, `CREATED` | "Заказ оформлен, готовится к отправке" |
| `ACCEPTED`, `SORTING` | "Заказ передан в службу доставки" |
| `IN_TRANSIT` | "Заказ в пути" |
| `ON_POINT`, `READY_FOR_PICKUP` | "Ожидает вас в пункте выдачи" |
| `ISSUED`, `DELIVERED` | "Заказ получен" |
| `RETURNING` | "Заказ не был забран, возвращается" |
| `RETURNED` | "Заказ возвращён отправителю" |
| `CANCELLED` | "Заказ отменён" |
| `LOST` | "Заказ утерян, свяжитесь с поддержкой" |

## Маппинг на наши статусы

| 5Post статус | ShipmentStatus | Наш OrderStatus |
|---|---|---|
| `NEW`, `CREATED` | `created` | `shipped` |
| `ACCEPTED` | `accepted` | `shipped` |
| `SORTING`, `IN_TRANSIT` | `in_transit` | `shipped` |
| `ON_POINT` | `on_point` | `shipped` |
| `READY_FOR_PICKUP` | `ready_for_pickup` | `ready_for_pickup` |
| `ISSUED`, `DELIVERED` | `issued` | `delivered` |
| `RETURNING` | `returning` | `client_dont_pickup` |
| `RETURNED` | `returned` | `returned_to_supplier` |
| `CANCELLED` | `cancelled` | `cancelled` |
| `LOST` | `lost` | `cancelled` |

## API эндпоинты

- `POST /api/v3/orders` — создание заказа (через партнёрский endpoint)
- `GET /api/v1/orders/{order_id}/status` — текущий статус + история трекинга
- `GET /api/v1/orders/{order_id}/label` — PDF-этикетка
- `DELETE /api/v3/orders/{order_id}` — отмена заказа
